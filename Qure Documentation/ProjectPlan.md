# Project Plan

## Project: Qure Clinic Management System

## 1. Project Overview

The Qure Clinic Management System is a web-based application designed to improve clinic access, appointment management, queue visibility, and clinic administration. The system supports different user roles, including visitors, patients, staff members, and administrators.

The project was completed over four development sprints, with each sprint focusing on a major area of functionality. The final project submission is due on **22 May**.

---

## 2. Project Objectives

The main objectives of the project are to:

- Allow visitors to access the landing page and navigate to login or sign-up.
- Allow patients to register, log in, search for clinics, book appointments, and manage their bookings.
- Allow patients to view and join clinic queues.
- Allow staff members to manage queues and assist patients with appointments.
- Allow administrators to manage clinics, staff, admins, operating hours, analytics, and system activity.
- Provide secure account management features such as password reset, username change, and password change.
- Provide clinic analytics and export functionality for reporting purposes.

# 8. Risks and Mitigation

| Risk | Impact | Mitigation |
|---|---|---|
| Authentication errors prevent users from logging in correctly. | High | Test all login roles and ensure users are redirected to the correct dashboard. |
| Appointment data does not update correctly after cancellation or rescheduling. | High | Test database updates and ensure the user interface refreshes after changes. |
| Queue positions become inaccurate. | High | Test joining, leaving, and staff queue updates carefully. |
| Analytics calculations are incorrect. | Medium | Compare displayed analytics with database values manually during testing. |
| Exported CSV or PDF files contain incorrect information. | Medium | Test exported files and check formatting, dates, and calculated values. |
| Deployment issues occur close to the deadline. | High | Test the deployed application before the final submission date. |
| Styling issues affect usability on different screen sizes. | Medium | Check the system on different screen sizes and fix major layout issues. |
