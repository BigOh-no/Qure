## Work Timeline and Contributions

For this sprint, my main contribution focused on integrating the backend functionality of the staff dashboard with the existing user interface. I worked on connecting the frontend components to Supabase( with the help of Vik) so that real-time clinic and queue information could be displayed correctly on the dashboard. I implemented functionality that retrieves clinic information linked to a staff member and displays the associated queue data dynamically on the page. This included fetching patient queue information, appointment-related data, and ensuring that the correct information was shown depending on the logged-in staff member. I also worked on improving the overall responsiveness and structure of the staff dashboard UI so that the backend data integrated cleanly into the existing design. I made a few changes to the original design since I felt that some of the elements were a bit unnecessary.


## Challenges Experienced
- One of the main difficulties I experienced was handling backend integration errors between the frontend and Supabase queries.
- I encountered issues where data was not rendering correctly due to incorrect query structures and mismatched database fields.
- I also faced styling issues after integrating dynamic data, as some UI components changed layout unexpectedly when real data was displayed.


## Reflection

Although debugging backend-related issues was time-consuming, I learnt a lot more about handling asynchronous operations, database queries, and state management within the application.
